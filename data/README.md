# data/

This directory contains both raw source data (downloaded from external providers or queried from public APIs) and processed outputs generated by the analysis notebooks. Raw data files are large and **not tracked by Git**. The cleaned boundaries and clustering outputs produced by the notebooks are committed, so the results can be inspected without re-running the full pipeline.

## Directory structure

```
data/
├── raw/                          # (gitignored) Source data, see acquisition instructions below
│   ├── fires.sqlite              # FPA FOD wildfire occurrence database (1992–2020)
│   └── trpa_boundary.geojson     # TRPA basin boundary (Lake Tahoe basin filter)
├── cached/                       # (gitignored) Pickled intermediates: fire projections and β sweeps
├── boundaries/                   # Cleaned lake shoreline polygons (tracked)
│   ├── tahoe_boundary.csv
│   └── mead_boundary.csv
├── processed/                    # Notebook outputs (tracked)
│   ├── tahoe_fires_clustered.csv
│   ├── mead_fires_clustered.csv
│   ├── mead_cluster_summary.csv
│   ├── mead_global_metrics.csv
│   └── mead_fires_nearshore_webmap.csv
└── README.md                     # This file
```

## Files tracked in Git

The following files are committed to the repository so that the clustering results can be reviewed without the raw inputs, and so `04_arcgis_dashboard.ipynb` can publish from `03_mead_wildfires_clustered.ipynb`'s outputs:

| File | Description |
|----|----|
| `boundaries/tahoe_boundary.csv` | Cleaned Lake Tahoe shoreline polygon, one row per vertex (`longitude`, `latitude`). Derived from NHD Waterbody layer 10 with the Tahoe Keys removed and the outline simplified. |
| `boundaries/mead_boundary.csv` | Cleaned Lake Mead shoreline polygon, one row per vertex (`longitude`, `latitude`). Derived from NHD Waterbody layer 12, unioning all features above 1 km² named "Lake Mead" (capturing the main body plus the eastern arm) and simplified with Douglas–Peucker. |
| `processed/tahoe_fires_clustered.csv` | All 1,068 Lake Tahoe basin fires with engineered features (`t_param`, `s_param`, `dist_to_lake_km`, `cause_binary`) and cluster labels from every method, at both scales: `cluster_basin_{std,eq,opt}` and `cluster_near_{std,eq,opt}` (near-shore labels are blank for fires outside the 1 km subset). |
| `processed/mead_fires_clustered.csv` | All 844 Lake Mead fires, same column structure as the Tahoe file (near-shore labels blank outside the 5 km subset). |
| `processed/mead_cluster_summary.csv` | Per-cluster statistics for the Lake Mead near-shore subset: 4 rows for standard k-means (`method = "std"`) and 4 for obstacle-aware optimized (`method = "oa_opt"`), with `n_fires`, mean and median fire size, percent human-caused and natural, and per-cluster arc-length span. Populates the dashboard side panels. |
| `processed/mead_global_metrics.csv` | Single-row headline metrics for the Lake Mead near-shore case: total fires, year range, `k`, optimal `β`, mean span for both methods, span improvement percent, and perimeter. Populates the dashboard headline indicators. |
| `processed/mead_fires_nearshore_webmap.csv` | Publishing-ready near-shore subset (214 fires within 5 km of shore) created by Notebook 04, with clean column names (`Cluster_Std`, `Cluster_OA`, `Cause`, `Fire_Size_Acres`, etc.) for upload to ArcGIS Online. |

## Raw data acquisition

To reproduce the analysis from scratch, obtain the following datasets and place them in the indicated directories. All raw files are gitignored.

### 1. FPA FOD wildfire occurrences

-   **Source:** Short, K. C. (2022). *Spatial Wildfire Occurrence Data for the United States, 1992–2020 (FPA_FOD)*, 6th Edition. USDA Forest Service Research Data Archive.
-   **URL:** <https://www.fs.usda.gov/rds/archive/catalog/RDS-2013-0009.6>
-   **Product:** SQLite database of U.S. wildfire records, 1992–2020.
-   **Placement:** Download the SQLite version, unzip it, and place (or rename) the database at `data/raw/fires.sqlite`. Notebooks 02 and 03 query the `Fires` table directly with bounding-box filters.

### 2. TRPA basin boundary

-   **Source:** Tahoe Regional Planning Agency, via Tahoe Open Data.
-   **URL:** <https://www.tahoeopendata.org/datasets/TRPA::trpa-boundary/about>
-   **Product:** TRPA Boundary (GeoJSON).
-   **Placement:** `data/raw/trpa_boundary.geojson`. Used by Notebook 02 to filter fires to the Lake Tahoe basin.

### 3. NHD lake boundaries

-   **Source:** USGS National Hydrography Dataset, queried live from the National Map map services.
-   **URL:** <https://hydro.nationalmap.gov/arcgis/rest/services/nhd/MapServer>
-   **Product:** NHD Waterbody features — Lake Tahoe (layer 10) and Lake Mead (layer 12).
-   **Placement:** No manual download required. Notebooks 02 and 03 query these layers at runtime and write the cleaned polygons to `data/boundaries/`. Requires an internet connection.

## Data use and ethical considerations

-   **Data governance:** FPA FOD, NHD, and the TRPA boundary are publicly available U.S. government datasets. The records describe wildfire locations and attributes and contain no personal or sensitive information.
-   **Spatial precision:** FOD ignition coordinates vary in precision, and some records are reported at coarse locations. Points are used as-is for descriptive clustering rather than parcel-level inference.
-   **Reporting bias:** The FOD compiles records from many federal, state, and local agencies, so reporting completeness varies across agencies, regions, and years. Spatial and temporal patterns should be read with this in mind.
-   **Intended use:** The clustering here is exploratory and descriptive. It is not a wildfire-risk model and should not be used for operational or planning decisions.
-   **Citation:** The FPA FOD archive (Short, 2022) should be cited in any reuse of these data.
